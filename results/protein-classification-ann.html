Protein Expression Classification in Mice

1. Introduction

This project aims to classify mice into eight groups based on protein expression measured in the nuclear fraction of the cortex. The groups are defined by genotype, behavioral condition, and treatment.

The main objective is to build and evaluate fully connected neural networks for multiclass classification. Two models were implemented:

  • Model 1, a baseline network with one hidden dense layer and dropout regularization.
  • Model 2, an extended architecture with additional complexity to explore potential performance improvements.

The comparison of these models allows assessing whether increased architectural complexity leads to better classification accuracy and generalization.

5. Split Dataset into Training and Testing Sets

The dataset was divided into training and testing sets to evaluate the models generalization performance. A total of 723 samples (approximately 2/3 of the data) were assigned to the training set, and 357 samples (approximately 1/3 of the data) were allocated to the testing set.

Both the feature matrices (F_train, F_test) and label vectors (L_train, L_test) were verified to contain no missing values, ensuring data integrity before model training.

The label distribution in both sets reflects all eight classes, confirming that each category is represented in the training and testing partitions. This split establishes a robust foundation for subsequent model training and validation.

6. Model 1: Fully Connected Neural Network

6.1. Model definition

A fully connected neural network (model_1) was designed for multiclass classification of mice based on protein expression. The architecture consists of:

  • Input layer: Takes 72 protein expression features.
  • Hidden layer: One dense layer with 35 neurons and ReLU activation, which introduces non-linearity and allows the network to learn complex feature interactions.
  • Dropout layer: 20% dropout is applied to the hidden layer to reduce overfitting by randomly deactivating neurons during training.
  • Output layer: 8 neurons with softmax activation, representing the eight possible classes.

The total number of trainable parameters is computed based on the layer dimensions:

  • Hidden layer: 35 neurons (72 inputs + 1 bias) = 2,555 parameters
  • Dropout layer: 0 parameters (only regularization)
  • Output layer: 8 neurons (35 inputs + 1 bias) = 288 parameters

Total parameters: 2,843, all of which are trainable.

This model provides a straightforward architecture suitable for small-to-medium datasets while incorporating regularization to improve generalization.

6.2. Model Training and Learning Curve

The first neural network model (model_1) was trained using a training set with an internal validation split of 20%. The network was compiled with the Adam optimizer and sparse_categorical_crossentropy loss function, which is appropriate for multiclass classification with integer labels. Training was performed over 50 epochs, allowing the model to iteratively adjust its weights to minimize the classification error.

Training Process

  • Labels were converted to integer type to ensure compatibility with the loss function.
  • The model was trained using the fit method with 50 epochs and a 20% validation split.
  • During training, both training and validation accuracy were monitored to evaluate learning progress.

The learning curve for model_1 shows both the training and validation accuracy over 50 epochs. Several observations can be made:

  • Initial phase (epochs 010): Both training and validation accuracy increase rapidly, indicating that the model is quickly learning basic patterns from the data.

  • Middle phase (epochs 1030): The growth rate of accuracy slows down, and the curves begin to fluctuate slightly. This is expected as the model fine-tunes its weights and balances between fitting the training data and generalizing to the validation set.

  • Later phase (epochs 3050): Validation accuracy consistently exceeds training accuracy, which is unusual but can occur due to the random nature of the validation split or dropout regularization. The fluctuations are moderate, and no clear sign of overfitting is observed.

  • Overall performance: The model reaches a final validation accuracy around 0.88, demonstrating good generalization to unseen data.

This plot confirms that model_1 is effectively learning the multiclass classification task without overfitting, and the dropout layer helps maintain a stable validation performance.

8. Model Comparison

A comparison between Model 1 and Model 2 on the test set shows the following:

  • Overall Accuracy: Model 1 achieved an accuracy of 0.75, while Model 2 reached 0.78, indicating a moderate improvement for Model 2 in generalization.

  • Class-wise Performance:

    • Both models perform very well on classes 2, 3, 4, 5, and 6. Model 2 generally achieves higher F1-scores in classes 1, 2, 3, 5, and 7.
    • Model 1 performs slightly better in class 4 (F1 0.91 vs. 0.84) and class 6 (F1 0.91 vs. 0.88).
    • Model 2 shows improved recall for classes 1, 3, and 5, which helps raise the overall accuracy.

  • Macro and Weighted Averages:

    • Macro average F1 improved from 0.76 (Model 1) to 0.78 (Model 2).
    • Weighted average F1 increased from 0.75 (Model 1) to 0.78 (Model 2), reflecting better performance on the most frequent classes.

Conclusion: Overall, Model 2 slightly outperforms Model 1, achieving higher accuracy and more balanced performance across classes. Model 1 remains strong in certain classes (especially class 4 and 6), but Model 2 demonstrates a better overall trade-off between precision and recall, making it the preferred choice for this multiclass classification task.
